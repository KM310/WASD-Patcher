# Contributing Guidelines

Thank you for your interest in contributing to this project!

## How to Contribute

### Reporting Issues
- Use the GitHub Issues tab.
- Describe the problem clearly.
- Include steps to reproduce the issue.
- Attach screenshots or logs if possible.

### Submitting Pull Requests
1. Fork the repository.
2. Create a new branch for your changes.
3. Make your edits and ensure the project still works.
4. Submit a Pull Request with a clear description of what you changed and why.

### Becoming an Official Contributor
If you want to become an official contributor or help maintain the project,  
**you must contact me on Discord: `k_m0310`**.

### Code Style
- Keep code clean and readable.
- Comment complex logic.
- Follow the structure already used in the project.

### Behavior
All contributors must follow the project's Code of Conduct.

Thank you for helping improve the project!
